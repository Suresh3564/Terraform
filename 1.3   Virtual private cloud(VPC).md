# 🚀 Terraform Installation & AWS Setup Guide

## 📌 1. Introduction to Terraform

Terraform is an **Infrastructure as Code (IaC)** tool developed by HashiCorp. It allows you to create, manage, and automate infrastructure (like servers, networks, databases) using code instead of manual configuration.

---

## 🌐 1.1 Creation of VPC (Detailed Explanation)

### 🔹 What is VPC?

A **VPC (Virtual Private Cloud)** is a logically isolated network in AWS where you can launch your resources like EC2, RDS, etc.

👉 Think of VPC as your **own private data center inside AWS**.

---

### 🔹 Why we create VPC?

* To isolate your infrastructure
* To control IP addressing (CIDR)
* To manage network security
* To design custom network architecture

# 🚀 AWS VPC Creation using Terraform (suresh-vpc)

## 📌 Project Overview

This project creates a complete AWS VPC infrastructure using Terraform.

It includes:

* VPC (suresh-vpc)
* 2 Public Subnets
* 2 Private Subnets
* Internet Gateway
* NAT Gateway
* Public & Private Route Tables

---

# 🏗️ Architecture

VPC CIDR: 10.0.0.0/16

Public Subnets:

* 10.0.1.0/24
* 10.0.2.0/24

Private Subnets:

* 10.0.3.0/24
* 10.0.4.0/24

# 📂 Terraform Code (Complete)

## 🔧 Provider

```hcl
provider "aws" {
  region = "ap-south-1"
}
```

---

## 🔧 VPC

```hcl
resource "aws_vpc" "suresh_vpc" {
  cidr_block = "10.0.0.0/16"

  tags = {
    Name = "suresh-vpc"
  }
}
```

---

## 🔧 Public Subnets

```hcl
resource "aws_subnet" "public_subnets" {
  count = 2

  vpc_id     = aws_vpc.suresh_vpc.id
  cidr_block = element(["10.0.1.0/24", "10.0.2.0/24"], count.index)

  availability_zone = "ap-south-1a"

  tags = {
    Name = "public-subnet-${count.index + 1}"
  }
}
```

---

## 🔧 Private Subnets

```hcl
resource "aws_subnet" "private_subnets" {
  count = 2

  vpc_id     = aws_vpc.suresh_vpc.id
  cidr_block = element(["10.0.3.0/24", "10.0.4.0/24"], count.index)

  availability_zone = "ap-south-1b"

  tags = {
    Name = "private-subnet-${count.index + 1}"
  }
}
```

---

## 🔧 Internet Gateway

```hcl
resource "aws_internet_gateway" "igw" {
  vpc_id = aws_vpc.suresh_vpc.id

  tags = {
    Name = "suresh-igw"
  }
}
```

---

## 🔧 Elastic IP

```hcl
resource "aws_eip" "nat_eip" {
  domain = "vpc"
}
```

---

## 🔧 NAT Gateway

```hcl
resource "aws_nat_gateway" "nat" {
  allocation_id = aws_eip.nat_eip.id
  subnet_id     = aws_subnet.public_subnets[0].id

  tags = {
    Name = "suresh-nat"
  }

  depends_on = [aws_internet_gateway.igw]
}
```

---

## 🔧 Public Route Table

```hcl
resource "aws_route_table" "public_rt" {
  vpc_id = aws_vpc.suresh_vpc.id

  route {
    cidr_block = "0.0.0.0/0"
    gateway_id = aws_internet_gateway.igw.id
  }

  tags = {
    Name = "public-route-table"
  }
}
```

---

## 🔧 Public Route Table Association

```hcl
resource "aws_route_table_association" "public_assoc" {
  count = 2

  subnet_id      = aws_subnet.public_subnets[count.index].id
  route_table_id = aws_route_table.public_rt.id
}
```

---

## 🔧 Private Route Table

```hcl
resource "aws_route_table" "private_rt" {
  vpc_id = aws_vpc.suresh_vpc.id

  route {
    cidr_block     = "0.0.0.0/0"
    nat_gateway_id = aws_nat_gateway.nat.id
  }

  tags = {
    Name = "private-route-table"
  }
}
```

---

## 🔧 Private Route Table Association

```hcl
resource "aws_route_table_association" "private_assoc" {
  count = 2

  subnet_id      = aws_subnet.private_subnets[count.index].id
  route_table_id = aws_route_table.private_rt.id
}
```

# ⚙️ How to Run

## 1. Initialize

terraform init

## 2. Validate

terraform validate

## 3. Plan

terraform plan

## 4. Apply

terraform apply -auto-approve

---

# 🌐 Workflow Explanation

1. VPC is created as the base network
2. Public subnets are created for internet-facing resources
3. Private subnets are created for backend services
4. Internet Gateway allows public internet access
5. NAT Gateway allows private subnets to access internet
6. Route Tables:

   * Public → IGW
   * Private → NAT

---
